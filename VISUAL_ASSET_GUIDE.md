# Visual Asset Guide for Prism AI

## Overview

This guide outlines the visual assets needed for the Prism AI launch. Since the application currently lacks a frontend UI, this guide serves as a blueprint for creating marketing materials and development priorities.

## Required Visual Assets

### 1. Application Screenshots

#### Dashboard/Home Page
- Clean, professional interface
- Document upload area with drag-and-drop functionality
- Recent documents list with previews
- User profile and subscription status
- Navigation menu

#### Document Upload Flow
- File selection screen with supported formats (PDF, DOCX, PPTX)
- Upload progress indicator with percentage
- Success confirmation with document preview
- Option to ask questions immediately

#### Query Interface
- Question input field with placeholder examples
- Smart suggestions as user types
- Results display with clear formatting
- Source citations for each answer
- Option to refine or rephrase question

#### Subscription/Upgrade Screen
- Comparison table of Explorer vs Pro plans
- M-Pesa payment integration with phone number input
- QR code for quick payment
- Benefits of upgrading to Pro tier

### 2. Marketing Materials

#### Product Hunt Banner
- Vertical format (800x600 pixels recommended)
- Eye-catching headline: "Your Personal Research Assistant"
- Key features highlighted with icons
- M-Pesa integration as unique selling point
- Call-to-action button

#### Social Media Assets

##### Twitter Header Image
- Horizontal format (1500x500 pixels)
- Clean, professional design
- Prism AI logo prominently displayed
- Tagline: "Your Personal Research Assistant"
- Call-to-action: "Try it now"

##### LinkedIn Banner
- Professional tone emphasizing productivity
- Target business professionals and researchers
- Highlight time-saving benefits
- Include relevant industry icons

##### Facebook Cover Photo
- Similar to Twitter header but optimized for Facebook
- Include social proof elements (testimonials, user count)
- Clear value proposition

### 3. Animated Demonstrations

#### Short GIF Demonstration (10-15 seconds)
1. Uploading a document (show progress bar)
2. Asking a question (typing animation)
3. Receiving an answer with sources (highlight citations)
4. Call-to-action to try the product

#### Explainer Video Script (60 seconds)

**Scene 1** (0-5 seconds)
- Person frustrated searching through documents
- Text overlay: "Spending hours searching through documents?"

**Scene 2** (5-15 seconds)
- Person uploads documents to Prism AI
- Smooth animation of documents being processed

**Scene 3** (15-35 seconds)
- Person asks a question
- Prism AI displays answer with source citations

**Scene 4** (35-50 seconds)
- Person happily working with insights from documents
- Text overlay: "Prism AI - Your Personal Research Assistant"

**Scene 5** (50-60 seconds)
- Prism AI logo and tagline
- Call-to-action: "Try Prism AI today"

### 4. Presentation Materials

#### Investor Pitch Deck
1. Cover slide with logo and tagline
2. Problem statement with statistics
3. Solution overview with diagrams
4. Key features and benefits
5. Technology stack
6. Market opportunity and size
7. Business model and revenue streams
8. Competitive advantage
9. Traction and milestones
10. Team introduction
11. Financial projections
12. Ask and next steps

#### Conference Presentation Template
- Consistent branding throughout
- Clean, readable fonts and colors
- Minimal text with visual aids
- Speaker notes section
- Contact information slide

## Design Guidelines

### Color Palette
- Primary: Professional blue (#2563EB)
- Secondary: Success green (#10B981)
- Accent: Warning orange (#F59E0B)
- Neutral: Light gray (#F3F4F6) and dark gray (#374151)

### Typography
- Headlines: Bold, sans-serif (e.g., Inter or Roboto)
- Body text: Readable sans-serif
- Monospace: For code snippets or technical information

### Iconography
- Consistent style (filled or outlined)
- Recognizable metaphors for actions
- Appropriate sizing for different contexts

## Quality Assurance

### Review Checklist
- [ ] Consistent branding across all assets
- [ ] Proper sizing for each platform
- [ ] Accessible color contrast
- [ ] Clear call-to-actions
- [ ] No spelling or grammatical errors
- [ ] Optimized file sizes
- [ ] Mobile-responsive designs where applicable

### Feedback Process
1. Internal review by team members
2. External review by beta testers
3. Iterate based on feedback
4. Final approval before public release
